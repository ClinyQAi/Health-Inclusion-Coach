# Contributing to Health Inclusion Coach

Thank you for your interest in contributing to the Health Inclusion Coach! This document provides guidelines for contributing to this project.

## ü§ù Code of Conduct

This project is dedicated to creating an inclusive environment. We expect all contributors to:

- Use welcoming and inclusive language
- Be respectful of differing viewpoints and experiences
- Accept constructive criticism gracefully
- Focus on what is best for the community

## üöÄ How to Contribute

### Reporting Issues

1. Check existing issues to avoid duplicates
2. Use the issue template when available
3. Provide clear, detailed descriptions
4. Include steps to reproduce bugs

### Submitting Changes

1. **Fork** the repository
2. **Create a branch** for your feature: `git checkout -b feature/your-feature-name`
3. **Make your changes** following our coding standards
4. **Test** your changes locally
5. **Commit** with clear, descriptive messages
6. **Push** to your fork
7. **Open a Pull Request** with a clear description

### Coding Standards

- Use TypeScript for all new code
- Follow existing code formatting (use Prettier if available)
- Write clear, self-documenting code with comments where needed
- Keep components small and focused
